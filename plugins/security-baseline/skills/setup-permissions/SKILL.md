# セットアップ: permissions テンプレート案内

このスキルはユーザーにプロジェクトの permissions テンプレートの存在を知らせ、適用の判断を委ねる。

## 手順

1. まず、ユーザーの現在のプロジェクトに `.claude/settings.json` が既に存在するか確認せよ。

2. 次に、このプラグインに同梱されている permissions テンプレートの内容を読み取れ:
   - パス: `${CLAUDE_SKILL_DIR}/../../templates/project-root/.claude/settings.json`
   - Read ツールでテンプレートの内容を取得する

3. ユーザーに以下を **表示** せよ（ファイルの書き換えは絶対にするな）:

   ### 表示する内容
   - このプラグインコレクション（forge-devkit）には **permissions テンプレート** が含まれていること
   - テンプレートに含まれる **deny ルール** の一覧（カテゴリ別に要約）
   - テンプレートに含まれる **allow ルール** の一覧
   - hooks（プラグイン）だけでは防げない攻撃経路があること:
     - `settings.json` の `deny` は **Read/Edit/Write ツールレベル** でブロックする
     - プラグインの hooks は **Bash ツール経由** の間接アクセスをブロックする
     - **両方あって初めて完全な防御になる**
   - 既に `.claude/settings.json` が存在する場合は、差分を示して「どの設定を取り込むか」を判断できるようにする

4. 最後に、ユーザーに選択肢を提示せよ:
   - **A**: テンプレートをそのままプロジェクトの `.claude/settings.json` に適用する（存在しない場合のみ）
   - **B**: 既存の settings.json にテンプレートの deny/allow を手動でマージする（diff を表示）
   - **C**: 今は適用しない（後で `/setup-permissions` で再実行できることを伝える）

## 重要な制約

- **ファイルを勝手に書き換えるな**。ユーザーが明示的に A または B を選択し、適用を承認した場合のみ書き込め。
- テンプレートの内容は表示するだけ。判断はユーザーに委ねろ。
- 既存の settings.json がある場合、上書きではなく **マージの提案** をせよ。
