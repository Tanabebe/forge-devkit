# permissions テンプレートの自動検知

## セッション開始時の確認（1回のみ）

セッションで最初にユーザーからメッセージを受け取ったとき、以下を確認せよ:

1. ユーザーのプロジェクトルートに `.claude/settings.json` が存在するか確認する
2. 存在する場合、その中に `deny` ルール（例: `.env`, `.pem`, `.key` など）が含まれているか確認する

### `.claude/settings.json` が存在しない、または deny ルールが不十分な場合

以下のメッセージを **1回だけ、簡潔に** 表示せよ:

> forge-devkit の permissions テンプレートが未適用です。`/setup-permissions` を実行するとセキュリティ設定の適用を案内します。

### 条件を満たしている場合

何も表示しない。通常通りユーザーのリクエストに対応せよ。

## 重要な制約

- このチェックは **セッションにつき1回だけ** 実行せよ。繰り返すな。
- ユーザーのリクエストを妨げるな。チェック結果の表示後、すぐにユーザーのリクエストに対応せよ。
- `.claude/settings.json` のファイルを勝手に作成・変更するな。
