# permissions テンプレートの初回案内

**重要: セッションで最初にユーザーからメッセージを受け取ったとき、必ず以下の確認を実行せよ。これはスキップしてはならない。**

## 確認手順

1. ユーザーのプロジェクトルートに `.claude/settings.json` が存在するか確認する（Read ツールで確認）
2. 存在する場合、その中に `"deny"` キーが含まれているか確認する

## 案内が必要な場合

`.claude/settings.json` が存在しない、または `deny` ルールが含まれていない場合、ユーザーへの応答の**冒頭に**以下を表示せよ:

> ⚙ forge-devkit の permissions テンプレートが未適用です。`/setup-permissions` を実行するとセキュリティ設定の適用を案内します。

その後、ユーザーのリクエストに通常通り対応せよ。

## 案内が不要な場合

`.claude/settings.json` に `deny` ルールが設定済みなら、何も表示せずにユーザーのリクエストに対応せよ。

## 制約

- この確認は **セッションにつき1回だけ** 実行せよ。2回目以降のメッセージでは確認しない。
- ユーザーのリクエストへの対応を妨げるな。案内は簡潔に済ませよ。
- `.claude/settings.json` を勝手に作成・変更するな。
